# Plan: v0.7.0 — OCR pre-fill

## Goal

After the user takes a label photo on the add-wine form, show a crop UI to select
the label region, then send the image to the Claude API to pre-fill producer, name,
vintage, and country. The user always reviews and corrects fields before submitting.

---

## Technology choice: Claude API (vision)

**Claude Haiku** (`claude-haiku-4-5-20251001`) via the Anthropic Messages API,
called directly from the browser using a user-supplied API key.

Alternatives considered and rejected:

| Alternative | Reason rejected |
|---|---|
| Tesseract.js (on-device WASM) | Poor accuracy on decorative fonts and multilingual labels |
| Florence-2 via Transformers.js | 500 MB+ model weights, unsuitable for a mobile PWA |
| Server proxy for Claude API | Adds infrastructure; breaks offline-first, single-user architecture |

The user supplies their own Anthropic API key in Settings. It is stored in
IndexedDB and sent only to `api.anthropic.com` over HTTPS. The
`anthropic-dangerous-direct-browser-access: true` header enables direct browser
calls without a proxy.

---

## Crop UI

Before OCR runs, a `CropBox` component overlays the photo and lets the user drag
a selection box to isolate the label. Two actions:

- **Crop & Read** — sends the cropped region to the Claude API
- **Skip** — sends the full (resized) image to the Claude API

The crop UI is independent of the OCR backend.

---

## Image normalisation

Before calling the API, all images are:

1. Decoded via `createImageBitmap`
2. Scaled down so the longest edge is ≤ 1568 px (Claude's recommended max)
3. Re-encoded as JPEG (or the original format if it is one of jpeg/png/gif/webp)

This keeps every request well under the 5 MB API limit, even for full-resolution
iPhone photos.

---

## Claude API call

Model: `claude-haiku-4-5-20251001`
Max tokens: 256
System prompt instructs the model to return **only** a JSON object:

```json
{"producer":"...","name":"...","vintage":"...","country":"..."}
```

The response is parsed with a regex to extract the JSON, then the fields are
applied to the form (only if the field is currently empty).

---

## API key settings

- Stored in IndexedDB under key `kellerverwaltung-settings-claude-key`
- Loaded on mount in both Settings and the Add Wine page
- Settings page: password input, no validation (optional — sync fields remain
  independent)
- If no key is set, the OCR pipeline still runs but will fail gracefully with an
  error note shown to the user
