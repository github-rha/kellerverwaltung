# v0.9.6 — Onboarding screen & filter dropdown fix

## Goal

Show a first-run onboarding screen once, and fix the filter dropdown going
off-screen by moving Filter + Sort to the left of the bottle image buttons.

---

## 1. Filter row reorder

`app/src/routes/+page.svelte` — change the single filter row from:

```
[red] [white] [sparkling] [dessert]  [Filter▼] [Sort▼]
```

to:

```
[Filter▼] [Sort▼]  [red] [white] [sparkling] [dessert]
```

The dropdown uses `absolute left-0 top-full`, which anchors to the left edge of
the `<div class="relative">` wrapper. With Filter first in the row, that wrapper
sits near the left edge of the screen, giving the dropdown plenty of room.

---

## 2. First-run onboarding

### Persistence

`app/src/lib/data/persist.ts` — add two functions using the existing idb-keyval
`get`/`set` imports:

```ts
const ONBOARDED_KEY = 'kellerverwaltung-onboarded'

export async function loadOnboarded(): Promise<boolean>
export async function markOnboarded(): Promise<void>
```

### UI

`app/src/routes/+page.svelte` — on mount, call `loadOnboarded()`. If `false`,
show a full-screen overlay (over the dashboard) until the user taps "Get
started", which calls `markOnboarded()` and hides the overlay.

Overlay content (kept minimal):

```
Kellerverwaltung

Track your wine cellar from your iPhone.

📱 Install: Open in Safari → Share → "Add to Home Screen"

🔄 Sync: Set up a private GitHub repo in Settings to back up
   your data and keep it safe.

[ Get started ]
```

The overlay is a fixed full-screen div with a white/light background, centred
content, and the wine accent colour on the button.
