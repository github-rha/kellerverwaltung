# v0.9.3 — Sync UX & inventory path

## Goal

Simplify the sync experience to a single Sync button with a meaningful safety
guard, move restore to Settings, and store dated inventory files in a directory.

## Changes

### 1. Inventory path: `cellar-list.txt` → `inventory/YYYY-MM-DD-cellar-list.txt`

`app/src/lib/data/sync.ts` — update `pushInventory` to write to
`inventory/${date}-cellar-list.txt` instead of `cellar-list.txt` in the repo
root. Each Inventory tap creates a new dated file; old files accumulate as a
history. The `date` string is passed in from the caller (already done this way).

### 2. Remove Push / Pull bar from dashboard

`app/src/routes/+page.svelte` — delete the entire sync bar block (Push button,
Pull button, unsynced indicator, pullBlocked warning, syncErrorMsg). Also remove
the related state variables and handler functions (`syncing`, `syncErrorMsg`,
`pullBlocked`, `handlePush`, `handlePull`, `handleForcePull`).

### 3. Add Sync button to dashboard header

`app/src/routes/+page.svelte` — add a Sync button in the header bar where
Import previously lived (between the settings gear and the add-wine bottle).

**Sync button mechanics:**
- Shown only when sync is configured; if not configured, show nothing (the
  "Set up sync →" prompt remains in the area below the filters)
- Disabled when offline or while syncing
- **Blocked if `wines.length < 10`**: shows an inline message below the header
  "Sync blocked — only {n} wines in local cellar" and does not push
- Otherwise: calls `push(settings)`, marks synced, shows "✓" briefly
- On error: shows error message inline below the header
- Unsynced dot (● unsynced / ● synced) shown next to the button

### 4. Restore from GitHub in Settings

`app/src/routes/settings/+page.svelte` — add a "Restore from GitHub" button
below the Inventory button:

- Only shown when sync is configured
- Always does a force-pull (no unsynced check — user is deliberately restoring)
- Requires `confirm()` dialog: "This will replace all local data with the remote
  version. This cannot be undone. Continue?"
- Disabled when offline or while restoring
- Shows "Restored ✓" on success, error message on failure

### 5. Remove pull-related exports no longer needed on the dashboard

`sync.ts` `pull()` (the unsynced-guarded version) is no longer called from the
dashboard. Keep it exported — `forcePull` is used by the new Restore button.
No deletions needed; the dashboard simply stops importing them.

## Acceptance criteria

- [ ] Inventory files are written to `inventory/YYYY-MM-DD-cellar-list.txt`
- [ ] Dashboard has no Push or Pull buttons
- [ ] Dashboard header has a Sync button when sync is configured
- [ ] Sync is disabled when offline
- [ ] Sync is blocked (with message) when local cellar has fewer than 10 wines
- [ ] Sync pushes successfully when ≥ 10 wines and online
- [ ] Unsynced indicator still visible on dashboard next to Sync button
- [ ] Settings has a "Restore from GitHub" button (sync configured only)
- [ ] Restore shows a confirmation dialog before proceeding
- [ ] Restore force-pulls without checking unsynced state
- [ ] `npm run lint`, `npm run format`, `npm run check`, `npm test` all pass
