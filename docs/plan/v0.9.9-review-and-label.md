# v0.9.9 — Code review fixes + quality label

## Goal
Fix three real issues found during external review, then sign off with the
haeusler-wein.ch quality label in Settings.

## Findings
1. `sortWines()` has no default return after the switch — returns undefined if
   an unexpected sort value is passed (noImplicitReturns is not in strict mode)
2. `claude.ts` has no try/catch around JSON.parse — uncaught exception if Claude
   returns unexpected output
3. `OcrResult.words` is dead code — always `[]`, never read anywhere

## Non-issues
- No XSS (Svelte escapes by default, no @html usage)
- Secrets correctly stored in IndexedDB, never in source
- Remote data validated via AJV before writing to local store
- No debug code, no unused imports

## Changes
- `app/src/lib/data/filter-sort.ts` — add default return in sortWines
- `app/src/lib/ocr/claude.ts` — try/catch JSON.parse; remove words field
- `app/src/routes/settings/+page.svelte` — add quality label
- `app/package.json` — bump to 0.9.9
