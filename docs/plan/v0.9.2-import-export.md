# v0.9.2 — Import & export

## Goal

Move CSV import out of the dashboard header into Settings (it's rarely used),
and add a one-tap export that pushes a print-ready plain-text inventory list
to the GitHub repo.

## Changes

### 1. Remove Import button from dashboard header

`app/src/routes/+page.svelte` — delete the `<a href="/import">Import</a>` link
from the header bar.

### 2. Add Import link to Settings

`app/src/routes/settings/+page.svelte` — add an "Import CSV →" row at the
bottom of the settings form that navigates to `/import`. Shown regardless of
sync configuration.

### 3. New export data function

New file `app/src/lib/data/export.ts`:

```
exportText(wines: WineEntry[], date: string): string
```

- Filters to wines with `bottles > 0`, sorted producer → name → vintage
- Produces a fixed-width plain-text table:

```
Keller — YYYY-MM-DD

Producer          Wine              Vintage  Bottles
─────────────────────────────────────────────────────
Domaine X         Riesling          2020           3
Weingut Y         Spätburgunder     2019           6

Total: 9 bottles
```

- Column widths are dynamic (max value width + 2 spaces padding, minimum 16 for
  Producer/Wine, 7 for Vintage, 7 for Bottles)
- Right-aligns Vintage and Bottles columns
- The separator line is as wide as the full row
- "NV" is used for non-vintage wines

### 4. Push export to GitHub

`app/src/lib/data/sync.ts` — add `pushExport(settings, text)` using the
existing `putFile`/`getFileMeta` helpers. Writes to `cellar-list.txt` in the
repo root. Commit message: `"export: update cellar list"`.

### 5. Inventory button in Settings

`app/src/routes/settings/+page.svelte` — add an "Inventory" button:

- Only shown when sync is configured (repo + PAT set)
- Disabled while export is in progress or offline
- On success shows a brief "Saved ✓" confirmation
- On error shows the error message

## File layout after this change
