# v0.9.1 — Bottle image filters and add button

## Goal

Replace text-only type filter pills and the "+" add button on the dashboard with
custom bottle PNG images already present in `app/static/`.

## Changes

### `app/src/routes/+page.svelte`

1. **Type filter row** (currently four text `<button>` pills — Red, White, Sparkling, Dessert):
   - Replace each button label with `<img src="/bottle-{type}.png" alt="{typeLabels[type]}" …>`
   - Keep the same toggle logic and active/inactive styling
   - Size images to fit comfortably as touch targets (≈ 40 × 40 px)

2. **Add-wine button** (currently a rounded `<a>` with text `+`):
   - Replace the `+` text with `<img src="/bottle-plus.png" alt="" …>`
   - Keep `aria-label="Add wine"` on the `<a>` element
   - Remove the fixed background color styling that was designed for the text glyph;
     let the image carry its own visual weight

## Acceptance criteria

- [ ] Tapping a bottle image toggles that type filter on/off
- [ ] Active image button shows a visible selected state (border or ring)
- [ ] Inactive image buttons are visually distinct from active state
- [ ] Active-filter chip still shows the type label text (no change)
- [ ] "Add wine" link navigates to `/wine/new`
- [ ] All images have appropriate `alt` text or the link has `aria-label`
- [ ] `npm run lint`, `npm run format`, `npm run check`, `npm test` all pass
